Interrupciones y Prioridades

DataCenter - Sistema de Control de Acceso y Monitoreo

1. Introducción

El sistema utiliza un Arduino Mega 2560 como maestro I2C que se comunica con tres esclavos (servidores, puertas, jardín) y un ESP32 como puente WiFi. Las interrupciones se manejan tanto a nivel hardware (I2C, sensores) como software (timers, comunicación serie).

3.1 Justificación de Prioridades

Crítica: El watchdog del ESP32 (10 segundos) asegura que el sistema se recupere automáticamente ante un congelamiento, evitando que los comandos de apertura de puertas queden sin procesar. La detección PIR se procesa con prioridad crítica porque una intrusión no detectada compromete la seguridad.

Alta: Las interrupciones I2C (onReceive/onRequest) tienen prioridad alta porque la comunicación entre el Mega y los esclavos es el canal principal de datos. Si se pierde una transmisión I2C, los datos de sensores se corrompen y los comandos de apertura no llegan a los servos.

Media: El control de servos (abrir/cerrar puertas) y el buzzer tienen prioridad media. Aunque importantes, no requieren respuesta instantánea; un retardo de milisegundos es aceptable.

Baja: Las lecturas de temperatura/humedad (DHT11) y los envíos HTTP al servidor pueden retrasarse sin afectar la funcionalidad principal del sistema.

5. Implementación Técnica

5.1 I2C onReceive (Esclavo 2)

void recibirComando(int bytes) {
  if (Wire.available()) {
    char cmd = Wire.read();
    if (cmd == 'O') {          // Open
      int puerta = Wire.read() - '0';
      if (puerta == 1 && !locked) abrirPuerta1();
      if (puerta == 2 && !locked) abrirPuerta2();
    } else if (cmd == 'C') {   // Close
      int puerta = Wire.read() - '0';
      if (puerta == 1) cerrarPuerta1();
      if (puerta == 2) cerrarPuerta2();
    } else if (cmd == 'L') {   // Lock
      locked = true;
    } else if (cmd == 'U') {   // Unlock
      locked = false;
    }
  }
}

5.2 Envío de Comando Lock desde el Mega

// Solo se activa por PIR (no por humo/temperatura)
if (alertaPuertas == 1 && !puertasLocked) {
  puertasLocked = true;
  Wire.beginTransmission(9);
  Wire.write('L');
  Wire.endTransmission();
} else if (alertaPuertas == 0 && puertasLocked) {
  puertasLocked = false;
  Wire.beginTransmission(9);
  Wire.write('U');
  Wire.endTransmission();
}

5.3 Buzzer Persistente (5 segundos)

if (alerta == 1 && !buzzerOn) {
  buzzerOn = true;
  buzzerStart = millis();
  digitalWrite(buzzerPin, HIGH);
}
if (buzzerOn && millis() - buzzerStart >= 5000) {
  buzzerOn = false;
  digitalWrite(buzzerPin, LOW);
}
if (alerta == 0) {
  buzzerOn = false;
  digitalWrite(buzzerPin, LOW);
}

5.4 Watchdog ESP32

#include "esp_task_wdt.h"

void setup() {
  esp_task_wdt_init(10, true);  // Timeout 10s
  esp_task_wdt_add(NULL);       // Vigilar tarea actual
}

void loop() {
  esp_task_wdt_reset();         // Reset watchdog cada ciclo
  // ... resto del código
}
Nota importante: El comando 'L' (lock) se envía exclusivamente cuando se detecta una intrusión por PIR (alertaPuertas == 1). En caso de incendio (humo == 1) o temperatura crítica, las puertas permanecen desbloqueadas para permitir la evacuación, cumpliendo con las normas de seguridad.
